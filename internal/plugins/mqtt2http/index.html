How to use

▼

This plugin listens to MQTT topics and triggers HTTP requests when conditions are met.

Trigger Conditions

  • Topic regex — regular expression to match MQTT topic, e.g. zigbee2mqtt/.*протечка.*
  • Payload regex — optional regex to match message payload, e.g. "water_leak":true
  • Mode:
    • Always — triggers on every matching message
    • On change — triggers only when payload match transitions from false to true (edge trigger). Use this for sensors that send data periodically — the action will fire once when the value changes, not on every report.
  • AND / OR — combine multiple conditions. OR = any condition fires, AND = all conditions must match simultaneously.

HTTP Action

  • Method — GET, POST, PUT, DELETE, PATCH
  • Body Type:
    • Forward MQTT payload — sends the raw MQTT JSON as request body
    • JSON / Form / Raw — custom body with template variable support
    • No body — sends request without body

Template Variables

Available in Body and URL fields (except "Forward MQTT payload" mode):

{{topic}} Full MQTT topic, e.g. zigbee2mqtt/Water Sensor
{{topic_name}} Last segment of topic (after last /), e.g. Water Sensor
{{payload}} Entire MQTT payload as string
{{payload.field}} Value of a specific JSON field, e.g. {{payload.battery}} → 100
{{timestamp}} Unix timestamp in milliseconds
{{timestamp_s}} Unix timestamp in seconds

Example: Water Leak → VK Notification

Topic regex: zigbee2mqtt/.*протечка.*
Payload regex: "water_leak":true
Mode: On change
Method: POST
Body Type: Form data
Body:
access_token=YOUR_TOKEN&peer_id=2000000002&message=🚨{{topic_name}}: протечка🚨&random_id={{timestamp}}&v=5.199
