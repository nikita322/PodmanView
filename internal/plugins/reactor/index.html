How to use

▼

Reactor listens to MQTT topics and executes a pipeline of actions when conditions are met.

Trigger

  • Topic regex — e.g. zigbee2mqtt/.*
  • Payload regex — e.g. "water_leak":true
  • Mode: Always (every message) or On change (edge trigger)

Action Pipeline

Actions execute sequentially. If any action fails, the pipeline stops.

  • HTTP — send an HTTP request (webhooks, APIs)
  • MQTT — publish to an MQTT topic (control Zigbee devices via Z2M)
  • Delay — pause before the next action

Template Variables

{{topic}}Full MQTT topic
{{topic_name}}Last segment of topic
{{payload}}Entire MQTT payload
{{payload.field}}JSON field value
{{timestamp}}Unix ms
{{timestamp_s}}Unix seconds

Example: Water Leak → Siren + VK + Auto-stop

Trigger: topic zigbee2mqtt/.*, payload "water_leak":true, mode: On change
Pipeline:
1. MQTT → zigbee2mqtt/sirena/set — {"warning":{"mode":"emergency"}}
2. HTTP → POST VK API — send notification
3. Delay → 60 seconds
4. MQTT → zigbee2mqtt/sirena/set — {"warning":{"mode":"stop"}}
